Lesson 2 - Landmark Elements

Introduction

In this lesson, you will learn about landmark elements in HTML. By the end of this lesson, you will understand how to structure your web pages using these important elements.

What are landmark elements?

When you create a web page, you want it to be accessible to as many people as possible. Landmark elements help achieve this by providing a clear structure to the content, making it easier to read and navigate. This helps you when you are writing your web page because it gives you a clear way to organise your content. To most people browsing your web site the landmark elements are invisible - the web page would look exactly the same if you left them out. But for people who need to use assistive technologies the landmark elements provide important information about the structure of the page and allow them to navigate it more easily. For example, a screen reader can use the landmark elements to quickly jump to the main content of the page, or to the navigation menu, without having to read through all the content in between.

Types of Landmark Elements

There are several types of landmark elements in HTML, each serving a specific purpose. The most common ones include:

  • <header>: This element represents the introductory content or a set of navigational links for a section of the page. It typically contains the site logo, title, and main navigation menu.
  • <nav>: This element is used to define a block of navigation links. It helps users understand that the links within this element are for navigating the site.
  • <main>: This element represents the main content of the document. It should be unique to the document and not contain any content that is repeated across pages (like sidebars or footers).
  • <footer>: This element represents the footer for its nearest sectioning content or sectioning root element. It typically contains information about the author, copyright information, or links to related documents.

Other landmark elements include:

  • <article> for self-contained content that could be distributed independently, such as a blog post or news article,
  • <aside> for tangential content that is related to the main content but not essential to its understanding, such as a sidebar with related links or advertisements, and
  • <section> for thematic grouping of content, which can be used to divide the main content into different sections with their own headings.
  • <figure> and <figcaption> for marking up images and their captions as a single unit.
