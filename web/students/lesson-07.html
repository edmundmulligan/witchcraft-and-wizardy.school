Lesson 7 - Firefox Developer Tools

Introduction

In this lesson, you will learn about Firefox Developer Tools, which are a set of powerful tools built into the Firefox web browser that allow you to inspect and debug your web pages. These tools are essential for any web developer, as they help you understand how your HTML code is being rendered by the browser and allow you to identify and fix issues in your code. You can use Firefox Developer Tools to view the structure of your HTML document, inspect the styles applied to different elements, and see how your page is laid out. They also include a console where you can see error messages and log information from your JavaScript code. In this lesson, you will learn how to open Firefox Developer Tools, navigate its different panels, and use it to inspect and debug your HTML code. This will be especially useful as you start working with css in the next lessons, since you will be able to see how your CSS styles are being applied to your HTML elements and make adjustments as needed.

If you are using a different browser, such as Chrome or Safari, you can still follow along with this lesson, as most browsers have similar developer tools. However, the specific steps to open and use the developer tools may vary slightly between browsers. If you are using a different browser, you can look up how to access the developer tools for that browser and then apply the same principles to inspect and debug your HTML code.

You will not be creating any new HTML code in this lesson. Instead, you will be using the developer tools to inspect and debug the HTML code you have already written in previous lessons. This is an important skill for any web developer, as it allows you to understand how your code is being rendered by the browser and identify any issues that may arise. By the end of this lesson, you should feel comfortable using Firefox Developer Tools to inspect and debug your HTML code, which will be a valuable skill as you continue to learn about web development.

Open Firefox Developer Tools

In this lesson we will focus on one tab: Inspector. We will not use the CSS rules area or Console yet, because you have not learned CSS or JavaScript at this stage.

You can open Firefox Developer Tools in several ways:

  • Press F12.
  • Press Ctrl + Shift + I (or Cmd + Option + I on Mac).
  • Right-click on the page and choose Inspect.

When the tools open, click the Inspector tab if it is not already selected.

How the Inspector Helps You Read HTML

The Inspector shows the HTML that the browser is currently rendering. Think of it as a live tree of your page structure.

  1. Open one of your earlier pages, for example http://localhost:8000/lesson-1.html.
  2. Open Developer Tools and choose the Inspector tab.
  3. Click the element picker icon (an arrow over a box), then click the heading on the page that says Hello, Witches and Wizards!.
  4. Firefox will highlight the matching HTML line, which should be the <h1> element.
  5. Expand and collapse nearby elements like <body> and <p> to see how tags are nested inside each other.

This is one of the fastest ways to answer: "Which HTML element created this bit of the page?"

Practice with Real Lesson HTML

Here is a short example from lesson 4. Use Inspector to find each tag in the live page and match it to what you see on screen. 

<section>
  <h2>A witch and Wizard</h2>
  <img src="rachel-mulligan-witch-old-female.png" width="200" alt="An old female witch">
  <img src="rachel-mulligan-wizard-old-male.png" width="200" alt="An old male wizard">
</section>
  1. Open http://localhost:8000/lesson-4.html.
  2. In Inspector, find the <main> element, then expand it.
  3. Find the first <section> and then its <h2> and <img> tags.
  4. Click each <img> tag in Inspector and check the src and alt attributes.

You are learning a very important debugging habit: compare what you expected to write with what the browser actually received.

How to Spot a Missing Image File

A very common HTML bug is an image that does not appear. One cause is a typo in the filename inside src.

For example, this line has a mistake:

<img src="rachel-mulligan-wizard-old-mael.png" width="200" alt="An old male wizard">

The file is really called rachel-mulligan-wizard-old-male.png, but this example uses mael instead of male.

  1. Open your page in Firefox and open the Inspector.
  2. Use the element picker and click where the missing image should be.
  3. Look at the selected <img> tag and read the src value carefully.
  4. Compare that filename to the real file in your project folder. If one letter is wrong, the browser cannot load the image.
  5. Fix the filename in your HTML file, save, and refresh the page.

Tip: if you can inspect the <img> element but no image appears, always check src, spelling, and file extension first.

Be curious

This is your chance to use Inspector like a detective tool. Open one of your earlier pages and explore what happens when you inspect different parts of it.

  • Open lesson-1.html and use the element picker to select the heading and paragraph. Then find the same <h1> and <p> tags in Inspector.
  • Open lesson-4.html and inspect each <img> element. Read the src and alt attributes and match them to the image shown on the page.
  • Intentionally create a typo in one image filename in your HTML (for example change male to mael), refresh the page, and use Inspector to find and fix the broken src value.
  • Expand and collapse elements in Inspector to trace the page structure from <body> to <main> to <section>.

You are not changing CSS or JavaScript yet. Focus on learning how to quickly locate HTML elements and check their attributes.

Conclusion

Congratulations. You have learned how to use Firefox Developer Tools to inspect HTML and debug common problems in your web pages.

In this lesson, you practiced opening the Inspector, selecting page elements, reading the HTML tree, and checking important attributes like src and alt. You also learned how to diagnose a missing image by carefully comparing the filename in your code with the real file name.

In the next lessons, this skill will become even more powerful when you begin using CSS and JavaScript, because Inspector will help you understand exactly what the browser is doing with your code.

Developer tools are now part of your everyday web wizard toolkit.
