Lesson 4 - Images

Introduction

In this lesson, you will learn about images in HTML. By the end of this lesson, you will understand how to add and manipulate images on your web pages.

What are images?

Up until now you have only worked with text on your web pages. There is a saying that “a picture is worth a thousand words”. It was probably an artist who said it rather than a writer, and while it is not true (sometimes the right words are more powerful than any image), images can greatly enhance the visual appeal and understanding of your content.

Types of Images

There are many different types of images that you can use on your web pages, such as photographs, illustrations, icons, and diagrams. There are also different image formats such as JPEG , PNG , GIF , SVG , and WebP . Each type and format has its own advantages and disadvantages, so it is important to choose the right one for your needs. For example, photographs are best saved in JPEG format because, although it uses lossy compression , it can compress the image without losing too much quality, while illustrations with transparent backgrounds are best saved in PNG format, which uses lossless compression . SVG is a vector image format that is ideal for logos and icons because it can displayed at any resolution without becoming pixelated . In the next section, you will learn how to add images to your web pages using HTML.

Creating a Web Page with Images

Before you can add images to your web page, you need to have some image files ready to use. There are two ways to do this: you can either create your own images or find images online that are free to use. Creating your own images gives you complete control over the content, but that is beyond the scope of this course. It is much easier to use images that are already available, although searching for exactly the right image for what you want to express can be time-consuming. You can find free images on websites like Unsplash, Pexels, and Pixabay. When you find an image you want to use, make sure to check the license to ensure that it is free to use for your purposes. Once you have your images, you will need to save them in the same folder as your HTML file or in a subfolder within that folder. This way, you can easily reference them in your HTML code.

For this lesson you can download the images you need from this web site. Browse to the Gallery page and when you find an image you want to use, right click on it and select "Save Image As..." to save it to your computer. You will need to save the images to your Projects folder where you are creating your web pages. You can also use your own images if you prefer. Just make sure to update the file names in the HTML code accordingly.

You will need a couple of portraits (PNG format), a couple of accessories (SVG format) and a couple of familiar photographs (JPEG format).

What does it mean

In this section, we will explain what the different parts of the HTML code you wrote mean.

  • Line 15: This line uses the <img> tag to display an image on the web page.
  • The src attribute specifies the path to the image file which you should change to match the location of your image file.
  • The width attribute specifies the width of the image in pixels . You can also specify a height attribute to set the height of the image - if you leave it out as we have done here the height will be calculated automatically. If you leave out width and height attributes, the picture will use its original dimensions, which may be too large for your web page. If you specify both, the image may be distorted to make it fit in the boundary you have specified.
  • The alt attribute provides alternative text for the image. This is used if the image cannot be displayed for some reason, such as if the file is missing or if the user is using a screen reader. Make sure the text in the alt attribute accurately describes the image.
  • Line 20: This is similar to the line 15 but uses an SVG image instead of a PNG. SVG images are scalable, meaning they can be resized without losing quality, making them ideal for logos and icons.
  • Lines 25-28: This lines display a JPEG image but the <img> tag on line 35 is wrapped in some extra HTML
  • Line 25: This is a <figure> element, which allow us to add a caption to the image.
  • Line 26: This is the <img> tag inside the <figure> element, which displays the image. It is similar to the previous <img> tags we have seen, except this time it uses a JPEG image.
  • Line 27: This is a <figcaption> element, which provides a caption for the image. This is different to the alt attribute on the image since the alt text is not normally visible while the figcaption is always displayed
