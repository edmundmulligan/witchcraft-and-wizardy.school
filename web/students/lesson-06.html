What are Tables?

Tables are a way to organise and display data in rows and columns. They are created using the <table> element in HTML. They are a bit more complex than other elements you have seen so far, since they require several other elements to define the structure of the table. The main elements used in tables are:

  • <table>: The container element for the entire table.
  • <tr>: Represents a table row.
  • <th>: Represents a header cell in a table, typically used for column or row headers.
  • <td>: Represents a standard data cell in a table.

Tables can be used to display all kinds of data, such as spells, potion ingredients, or any other structured data that benefits from being organised in rows and columns.

What does it mean

In this section, we will explain what the different parts of the HTML code you wrote mean.

  • Line 15: This is where the table starts. The <table> element is the container for the entire table. It tells the browser that everything inside it is part of the table structure. The attribute border="1" is used to add a simple border around the table cells, which makes it easier to see the structure of the table. Later on you will learn how to use CSS to style your tables instead of using the border attribute, but this is a simple way to add some basic styling for now. Note that you do NOT need to say how many rows and columns the table has in the <table> element. The browser will figure this out based on the number of <tr>, <th>, and <td> elements you include inside the table.
  • Line 16: This is an optional caption for the table. The <caption> element is used to provide a title or description for the table. It is typically displayed above the table, and can be useful for providing context about the data being displayed.
  • Line 17: This is where the table header starts. The <thead> element is used to group the header content in a table. It helps to distinguish the header from the body of the table. It is optional, but it can be useful for styling and accessibility purposes.
  • Line 18: This is where a table row starts. The <tr> element is used to define a row in the table. Each row can contain header cells (<th>) or data cells (<td>), depending on the structure of your table.
  • Lines 19-21: These lines define the header cells for the table. The <th> element is used to define a header cell, which is typically displayed in bold and centred by default. In this example, we have three header cells: "Titles", "Witch ", and "Wizard". These headers describe the type of data that will be displayed in each column of the table. Because there are three <th> elements in this row, the browser will understand that the table has three columns.
  • Lines 22-23: These lines are the closing tags for the table header section. The </tr> closes the table row, and the </thead> tag indicates the end of the header section.
  • Line 24: This is where the table body starts. The <tbody> element is used to group the body content in a table. It helps to distinguish the body from the header of the table. Like <thead>, it is optional, but it can be useful for styling and accessibility purposes.

In this example, the footer row only has two cells, but the second cell uses the attribute colspan="2" to indicate that it should span across two columns. This allows us to have a footer row with fewer cells than the header and body rows, while still maintaining the overall structure of the table.
  • Lines 58-62: This cell has two hyperlinks in it, which demonstrate how you can include links to other web pages or resources within a table cell.
  • Line 60: This uses the <br> element to create a line break between the two hyperlinks, so that they appear on separate lines within the same cell. The <br> element is an empty element that does not require a closing tag, and it is used to insert a line break in the content. In this case, it helps to improve the readability of the links in the footer cell by separating them onto different lines. We could have instead created two rows in the footer, each with one hyperlink, and then used the rowspan attribute to have the header cell span both rows, but using a line break is a simpler way to achieve the same result in this case.

  • Note how the different elements of the table have been indented. This is not required, but it helps to make the structure of the table clearer and easier to read. It is a good practice to use consistent indentation in your HTML code to help you and other people understand the structure of your code.
