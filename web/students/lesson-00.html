Lesson 0 - Getting Started

Introduction

In this lesson, you will ensure that you have all the necessary tools installed on your computer to start learning web development. By the end of this lesson, you will be ready to create your first web page. If you are at a Code Dojo and are using a club computer, these tools will already be installed for you so you can skip to the next lesson.

Installing Required Software

These are the tools you need to install. Some may already be installed on your computer, in which case you can skip those sectios. You may use other tools if you prefer, but these are the ones the lessons will assume you have.

  • Microsoft Visual Studio Code (or another code editor).
  • Mozilla Firefox (or another modern web browser such as Google Chrome or Apple Safari).
  • Git (optional, for version control, used in later lessons).
  • PHP (optional, for server-side scripting, used in later lessons).
  • SQLite (optional, for database management, used in later lessons).

If you don't want to install Git, PHP, or SQLite right now, that's okay; you can install them later when you get to the lessons that need them.

Please select your operating system below to see the specific installation instructions.

What computer do you use?

Windows Installation Instructions

The latest version of Windows is Windows 11, but you should be able to use Windows 10 as well. The lessons have not been tested on earlier versions of Windows, but they may work ok. It's possible that the browser on older versions of windows won't run all the features used in the lessons. In particular, beware of using Internet Explorer - it is not supported anymore and Microsoft Edge is recommended instead. Here are the instructions for installing the recommended software on your Windows computer:

Visual Studio Code

Check whether Visual Studio Code is already installed:

  • Press Windows key + R
  • Type code and press Enter
  • If VS Code opens, you have it installed. If you get an error message, you need to install it.

If you need to install Visual Studio Code, follow these steps:

  1. Download VS Code from the Official Website
    • Open your web browser
    • Go to https://code.visualstudio.com
    • Click the large blue "Download for Windows" button
    • The file VSCodeUserSetup-x64-[version].exe will download (usually to your Downloads folder)
  2. Run the Installer
    • Open your Downloads folder (press Windows key + E, then click Downloads)
    • Double-click the VSCodeUserSetup file you just downloaded
    • If Windows asks "Do you want to allow this app to make changes?", click Yes
  3. Setup Wizard
    • Accept the license agreement, then click Next
    • Choose installation location (default is fine), click Next
    • Choose Start Menu folder (default is fine), click Next
  4. Select Additional Tasks
    • Check these recommended options:
      • Create a desktop icon
      • Add "Open with Code" action to Windows Explorer file context menu
      • Add "Open with Code" action to Windows Explorer directory context menu
      • Register Code as an editor for supported file types
      • Add to PATH
    • Click Next
  5. Install
    • Review your choices, then click Install
    • Wait for installation to complete (takes about 1-2 minutes)
    • Check "Launch Visual Studio Code", then click Finish
  6. First Launch
    • VS Code opens with a Welcome screen
    • You can choose your color theme (light or dark)
    • You're ready to start coding!
Mozilla Firefox

Check whether Firefox is already installed:

  • Press Windows key
  • Type Firefox
  • If Firefox appears in the search results, you have it installed. If not, you need to install it.

If you need to install Firefox, follow these steps:

  1. Download Firefox from the Official Website
    • Open your web browser (Microsoft Edge or another browser)
    • Go to https://www.mozilla.org/firefox/
    • Click the blue "Download Firefox" button
    • The file Firefox Installer.exe will download (usually to your Downloads folder)
  2. Run the Installer
    • Open your Downloads folder (press Windows key + E, then click Downloads)
    • Double-click the Firefox Installer.exe file you just downloaded
    • If Windows asks "Do you want to allow this app to make changes?", click Yes
  3. Installation Process
    • The installer will start automatically
    • Firefox will install to the default location (usually C:\Program Files\Mozilla Firefox)
    • Wait for installation to complete (takes about 30 seconds to 1 minute)
    • Firefox will automatically launch when installation is complete
  4. First Launch
    • Firefox opens with a Welcome screen
    • You can choose to set Firefox as your default browser (optional)
    • You can import bookmarks and settings from other browsers (optional)
    • You're ready to browse!
  5. Pin to Taskbar (Optional)
    • With Firefox running, right-click the Firefox icon in the taskbar
    • Select "Pin to taskbar"
    • Now you can launch Firefox quickly from the taskbar
  6. Verify Installation
    • In Firefox, click the menu button (three horizontal lines) in the top-right corner
    • Click "Help" → "About Firefox"
    • You should see the Firefox version number
Git

Check whether Git is already installed:

  • Press Windows key + R
  • Type cmd and press Enter to open Command Prompt
  • Type git --version and press Enter
  • If you see a version number, you have it installed. If you get an error message, you need to install it.

If you need to install Git, follow these steps:

  1. Download Git from the Official Website
    • Open your web browser
    • Go to https://git-scm.com/
    • Click the "Download for Windows" button
    • The file Git-[version]-64-bit.exe will download (usually to your Downloads folder)
  2. Run the Installer
    • Open your Downloads folder (press Windows key + E, then click Downloads)
    • Double-click the Git installer file you just downloaded
    • If Windows asks "Do you want to allow this app to make changes?", click Yes
  3. Setup Wizard - License
    • Read the GNU General Public License
    • Click Next
  4. Setup Wizard - Installation Location
    • Default location is fine (C:\Program Files\Git)
    • Click Next
  5. Setup Wizard - Select Components
    • Keep the default selections (recommended):
      • Additional icons (On the Desktop)
      • Git Bash Here
      • Git GUI Here
      • Associate .git* configuration files
      • Associate .sh files to be run with Bash
    • Click Next
  6. Setup Wizard - Start Menu Folder
    • Default is fine ("Git")
    • Click Next
  7. Setup Wizard - Default Editor
    • Select "Use Visual Studio Code as Git's default editor" from the dropdown
    • Click Next
  8. Setup Wizard - Path Environment
    • Select "Git from the command line and also from 3rd-party software" (recommended)
    • Click Next
  9. Setup Wizard - Remaining Options
    • For all remaining screens, keep the default settings
    • Click Next through each screen
    • Finally, click Install
    • Wait for installation to complete (takes about 1-2 minutes)
    • Uncheck "View Release Notes" and click Finish
  10. Configure Git
    • Press Windows key + R
    • Type cmd and press Enter
    • Set your name (replace "Your Name" with your actual name): 
      git config --global user.name "Your Name"
    • Set your email (replace with your actual email): 
      git config --global user.email "your.email@example.com"
    • These settings will be used to identify you in your Git commits
  11. Verify Installation
    • In the same Command Prompt, type: 
      git --version
    • You should see the Git version number
    • Check your configuration: 
      git config --global --list
    • You should see your name and email listed
PHP

Check whether PHP is already installed:

  • Press Windows key + R
  • Type cmd and press Enter to open Command Prompt
  • Type php --version and press Enter
  • If you see a version number, you have it installed. If you get an error message, you need to install it.

If you need to install PHP, follow these steps:

  1. Download PHP from the Official Website
    • Open your web browser
    • Go to https://windows.php.net/download/
    • Under "Current Stable PHP", find the latest version
    • Download the Non-Thread Safe (NTS) ZIP file for x64 (64-bit)
      • Look for something like "VS16 x64 Non Thread Safe" ZIP
      • Non-Thread Safe is recommended for most users and works with PHP's built-in server
    • The file will be named something like php-8.x.x-nts-Win32-vs16-x64.zip
  2. Extract PHP
    • Open your Downloads folder
    • Right-click the PHP ZIP file you downloaded
    • Select "Extract All..."
    • Change the destination to C:\php (create this folder if needed)
    • Click "Extract"
    • You should now have PHP files in C:\php
  3. Add PHP to System PATH
    • Press Windows key and type "environment variables"
    • Click "Edit the system environment variables"
    • Click the "Environment Variables" button at the bottom
    • In the "System variables" section, find and select "Path"
    • Click "Edit"
    • Click "New"
    • Type C:\php and press Enter
    • Click "OK" on all windows to save
  4. Configure PHP (Basic Setup)
    • Open File Explorer and go to C:\php
    • Find the file php.ini-development
    • Right-click it and select "Copy"
    • Right-click in the same folder and select "Paste"
    • Rename the copy to php.ini (remove the "-development" part)
    • This creates your PHP configuration file
  5. Enable Common Extensions (Optional)
    • Right-click php.ini and open it with Notepad or VS Code
    • Find these lines (use Ctrl+F to search) and remove the semicolon (;) at the start: 
      ;extension=curl
;extension=fileinfo
;extension=gd
;extension=mbstring
;extension=mysqli
;extension=openssl
;extension=pdo_mysql
;extension=pdo_sqlite
    • After removing semicolons, they should look like: 
      extension=curl
extension=fileinfo
      (and so on...)
    • Save the file and close it
  6. Restart Command Prompt and Verify
    • Close any open Command Prompt windows
    • Press Windows key + R, type cmd, press Enter
    • Type: 
      php --version
    • You should see the PHP version number
    • Test PHP by running: 
      php -r "echo 'PHP is working!';"
    • You should see "PHP is working!" printed
  7. Test PHP Built-in Server (Optional)
    • Create a test folder: mkdir C:\test-php
    • Navigate to it: cd C:\test-php
    • Create a test file: 
      echo ^<?php phpinfo(); ?^> > index.php
    • Start the PHP server: 
      php -S localhost:8000
    • Open your browser and go to http://localhost:8000
    • You should see the PHP information page
    • Press Ctrl + C in Command Prompt to stop the server
SQLite

Check whether SQLite is already installed:

  • Press Windows key + R
  • Type cmd and press Enter to open Command Prompt
  • Type sqlite3 --version and press Enter
  • If you see a version number, you have it installed. If you get an error message, you need to install it.

If you need to install SQLite, follow these steps:

  1. Download SQLite from the Official Website
    • Open your web browser
    • Go to https://www.sqlite.org/download.html
    • Scroll down to the "Precompiled Binaries for Windows" section
    • Download two files:
      • sqlite-tools-win32-x86-*.zip (contains sqlite3.exe command-line tool)
      • sqlite-dll-win64-x64-*.zip (contains the DLL library)
    • Both files will download to your Downloads folder
  2. Create SQLite Directory
    • Press Windows key + E to open File Explorer
    • Navigate to your C: drive
    • Create a new folder called sqlite (full path: C:\sqlite)
  3. Extract SQLite Files
    • Go to your Downloads folder
    • Right-click sqlite-tools-win32-x86-*.zip
    • Select "Extract All..."
    • Change destination to C:\sqlite and click Extract
    • Right-click sqlite-dll-win64-x64-*.zip
    • Select "Extract All..."
    • Change destination to C:\sqlite and click Extract
    • Move all .exe and .dll files from the subfolders directly into C:\sqlite
  4. Add SQLite to System PATH
    • Press Windows key and type "environment variables"
    • Click "Edit the system environment variables"
    • Click the "Environment Variables" button at the bottom
    • In the "System variables" section, find and select "Path"
    • Click "Edit"
    • Click "New"
    • Type C:\sqlite and press Enter
    • Click "OK" on all windows to save
  5. Verify Installation
    • Close any open Command Prompt windows
    • Press Windows key + R, type cmd, press Enter
    • Type: 
      sqlite3 --version
    • You should see the SQLite version number
  6. Test SQLite (Optional)
    • In Command Prompt, type: 
      sqlite3
    • You should see the SQLite prompt: sqlite>
    • Try a simple command: 
      SELECT 'SQLite is working!';
    • You should see the message printed
    • Type .quit and press Enter to exit SQLite
  7. Create a Test Database (Optional)
    • In Command Prompt, navigate to a test folder: 
      mkdir C:\test-sqlite
cd C:\test-sqlite
    • Create a database: 
      sqlite3 test.db
    • Create a simple table: 
      CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT);
INSERT INTO users (name) VALUES ('Alice');
SELECT * FROM users;
    • You should see the data you inserted
    • Type .quit to exit
Visual Studio Code Extensions

VS Code extensions add extra features and functionality to your editor. Here are the recommended extensions for web development:

How to Install Extensions
  1. Open Extensions Panel
    • Open VS Code
    • Press Ctrl + Shift + X to open the Extensions panel, or
    • Click the Extensions icon in the left sidebar (looks like four squares)
  2. Search and Install
    • Type the extension name in the search box
    • Click the "Install" button for the extension you want
    • Wait for installation to complete
    • Some extensions may require VS Code to reload
Essential Extensions (Install Now)
  1. Live Server (ritwickdey.LiveServer)
    • Launches a local development server with live reload
    • Your browser automatically refreshes when you save changes
    • Essential for testing HTML/CSS/JavaScript
    • To use: Right-click an HTML file and select "Open with Live Server"
  2. Prettier - Code formatter (esbenp.prettier-vscode)
    • Automatically formats your code consistently
    • Makes your code easier to read
    • After installing, you can format by pressing Shift + Alt + F
    • Enable "Format On Save" in settings for automatic formatting
  3. Auto Rename Tag (formulahendry.auto-rename-tag)
    • Automatically renames paired HTML/XML tags
    • When you rename an opening tag, the closing tag updates too
    • Saves time and prevents errors
  4. HTML CSS Support (ecmel.vscode-html-css)
    • Provides better autocomplete for HTML class and ID attributes
    • Suggests CSS classes from your stylesheets
    • Makes writing HTML faster and more accurate
Helpful Extensions
  1. Path Intellisense (christian-kohler.path-intellisense)
    • Autocompletes file paths in your code
    • Helpful when linking CSS files, images, or other resources
  2. Indent-Rainbow (oderwat.indent-rainbow)
    • Colors indentation levels with subtle colors
    • Makes code structure more visible
    • Helps you spot indentation errors quickly
  3. Error Lens (usernamehw.errorlens)
    • Shows errors and warnings inline in your code
    • Makes debugging easier by highlighting problems immediately
For Later Lessons

Install these when you reach the relevant lessons:

  1. PHP Intelephense (bmewburn.vscode-intelephense-client)
    • PHP language support with autocomplete and error checking
    • Install when you start learning PHP
  2. SQLite Viewer (qwtel.sqlite-viewer)
    • View and query SQLite databases directly in VS Code
    • Install when you start working with databases
Optional Extensions
  1. Better Comments (aaron-bond.better-comments)
    • Color-codes different types of comments (TODO, FIXME, etc.)
    • Makes comments more organized and noticeable
  2. axe Accessibility Linter (deque-systems.vscode-axe-linter)
    • Checks HTML for accessibility issues
    • Helps you build websites that everyone can use
Configure Prettier (Recommended)

To make Prettier format your code automatically when you save:

  1. Press Ctrl + , to open Settings
  2. Search for "format on save"
  3. Check the box for "Editor: Format On Save"
  4. Search for "default formatter"
  5. Select "Prettier - Code formatter" from the dropdown

macOS Installation Instructions

The latest version of Apple's macOS is macOS Ventura, but you should be able to use recent versions as well. The lessons have not been tested on earlier versions of macOS, but they may work ok. Here are the instructions for installing the recommended software on your macOS computer:

Visual Studio Code

Check whether Visual Studio Code is already installed:

  • Open Finder
  • Go to the Applications folder
  • If you see Visual Studio Code.app, you have it installed. If not, you need to install it.

If you need to install Visual Studio Code, follow these steps:

  1. Download VS Code from the Official Website
    • Open your browser
    • Go to https://code.visualstudio.com
    • Click the large blue "Download for Mac" button
    • The file VSCode-darwin-universal.zip will download (usually to your Downloads folder)
  2. Extract the Application
    • Open Finder and go to your Downloads folder
    • Double-click the VSCode-darwin-universal.zip file to extract it
    • You'll see a Visual Studio Code.app file appear
  3. Move to Applications Folder
    • Drag Visual Studio Code.app from Downloads to your Applications folder
    • You can open Applications by pressing Command + Shift + A in Finder
    • Or drag it to the Applications icon in the Finder sidebar
    • This installs VS Code permanently on your Mac
  4. First Launch
    • Open your Applications folder
    • Double-click Visual Studio Code
    • macOS may show a security warning: "Visual Studio Code is an app downloaded from the Internet. Are you sure you want to open it?"
    • Click Open
  5. Add to Dock (Optional)
    • While VS Code is running, right-click its icon in the Dock
    • Select Options → Keep in Dock
    • Now you can launch it quickly from the Dock
  6. Install Command Line Tools (Recommended)
    • In VS Code, press Command + Shift + P to open the Command Palette
    • Type shell command
    • Select "Shell Command: Install 'code' command in PATH"
    • Enter your password if prompted
    • Now you can open VS Code from Terminal by typing code
  7. Verify Installation:
    • Open Terminal (Command + Space, type "Terminal")
    • Type code --version and press Enter
    • You should see the VS Code version number

PHP

Check whether PHP is already installed:

  • Open Terminal (Command + Space, type "Terminal")
  • Type php --version and press Enter
  • If you see a version number, you have PHP installed. Note: macOS may come with an older PHP version pre-installed.

Note: Recent versions of macOS (Monterey and later) no longer include PHP by default, so you'll need to install it.

Install PHP via Homebrew (Recommended)

Homebrew is the easiest way to install and manage PHP on macOS.

  1. Install Homebrew (if not already installed)
    • Open Terminal
    • Check if Homebrew is installed by typing: 
      brew --version
    • If you get "command not found", install Homebrew with: 
      /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
    • Follow the on-screen instructions
    • You may need to add Homebrew to your PATH (the installer will tell you how)
  2. Install PHP
    • Once Homebrew is installed, run: 
      brew install php
    • This will install the latest stable version of PHP
    • Wait for installation to complete (may take a few minutes)
  3. Verify Installation
    • Close and reopen Terminal
    • Type: 
      php --version
    • You should see the PHP version number (should be 8.x)
    • Check where PHP is installed: 
      which php
    • Should show something like /opt/homebrew/bin/php or /usr/local/bin/php
  4. Test PHP
    • Run a simple PHP command: 
      php -r "echo 'PHP is working!';"
    • You should see "PHP is working!" printed
  5. Test PHP Built-in Server (Optional)
    • Create a test directory: 
      mkdir ~/test-php && cd ~/test-php
    • Create a test file: 
      echo '<?php phpinfo(); ?>' > index.php
    • Start the PHP server: 
      php -S localhost:8000
    • Open your browser and go to http://localhost:8000
    • You should see the PHP information page
    • Press Control + C in Terminal to stop the server
Update PHP Configuration (Optional)
  • To find your php.ini location: 
    php --ini
  • Edit the configuration file if needed: 
    code $(php --ini | grep 'Loaded Configuration File' | cut -d ':' -f 2 | xargs)
SQLite

Check whether SQLite is already installed:

  • Open Terminal (Command + Space, type "Terminal")
  • Type sqlite3 --version and press Enter
  • If you see a version number, you have it installed.

Good News: macOS comes with SQLite pre-installed! You usually don't need to install it separately.

Verify Existing Installation
  1. Check SQLite Version
    • Open Terminal
    • Type: 
      sqlite3 --version
    • You should see the SQLite version (typically 3.x.x)
  2. Test SQLite
    • Type: 
      sqlite3
    • You should see the SQLite prompt: sqlite>
    • Try a simple command: 
      SELECT 'SQLite is working!';
    • You should see the message printed
    • Type .quit and press Enter to exit SQLite
Update SQLite via Homebrew (Optional)

If you want a newer version than what comes with macOS:

  1. Install/Update via Homebrew
    • Open Terminal
    • If you have Homebrew installed: 
      brew install sqlite3
    • Or to update an existing installation: 
      brew upgrade sqlite3
  2. Use the Homebrew Version
    • The Homebrew version will be installed to /opt/homebrew/bin/sqlite3 or /usr/local/bin/sqlite3
    • Check which version is being used: 
      which sqlite3
    • To prefer the Homebrew version, you may need to adjust your PATH in your shell configuration
Create a Test Database (Optional)
  1. Create a Test Directory
    • In Terminal, run: 
      mkdir ~/test-sqlite && cd ~/test-sqlite
  2. Create a Database
    • Run: 
      sqlite3 test.db
    • Create a simple table: 
      CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT);
INSERT INTO users (name) VALUES ('Alice');
SELECT * FROM users;
    • You should see the data you inserted
    • Type .quit to exit
  3. View the Database File
    • List files in the directory: 
      ls -lh
    • You should see test.db

Linux Installation Instructions

There are many different versions of Linux available, so these instructions will cover the most common distributions: Ubuntu and Fedora. If you are using a different distribution, the steps may vary slightly. The lessons have been tested on Linux Mint 22.2 (which is based on Ubuntu 22.04) but should work fine with other flavours and versions. Here are the instructions for installing the recommended software on your Linux computer:

Visual Studio Code

Check whether Visual Studio Code is already installed:

  • Open Terminal (usually Ctrl + Alt + T)
  • Type code --version and press Enter
  • If you see a version number, you have it installed. If you get a "command not found" error, you need to install it.
Ubuntu / Debian-based Linux

If you need to install Visual Studio Code on Ubuntu, follow these steps:

  1. Download and Import Microsoft GPG Key
    • Open Terminal
    • Run this command to download and install the Microsoft GPG key: 
      wget -qO- https://packages.microsoft.com/keys/microsoft.asc | gpg --dearmor > packages.microsoft.gpg
sudo install -D -o root -g root -m 644 packages.microsoft.gpg /etc/apt/keyrings/packages.microsoft.gpg
  2. Add VS Code Repository
    • Run this command to add the VS Code repository: 
      sudo sh -c 'echo "deb [arch=amd64,arm64,armhf signed-by=/etc/apt/keyrings/packages.microsoft.gpg] https://packages.microsoft.com/repos/code stable main" > /etc/apt/sources.list.d/vscode.list'
  3. Update Package List and Install
    • Update your package list: 
      sudo apt update
    • Install VS Code: 
      sudo apt install code
    • Wait for installation to complete
  4. Launch VS Code
    • Type code in Terminal and press Enter, or
    • Find "Visual Studio Code" in your application menu
    • Click to open VS Code
Fedora / Red Hat-based Linux

If you need to install Visual Studio Code on Fedora, follow these steps:

  1. Import Microsoft GPG Key
    • Open Terminal
    • Run this command: 
      sudo rpm --import https://packages.microsoft.com/keys/microsoft.asc
  2. Add VS Code Repository
    • Run this command to create the repository file: 
      sudo sh -c 'echo -e "[code]\nname=Visual Studio Code\nbaseurl=https://packages.microsoft.com/yumrepos/vscode\nenabled=1\ngpgcheck=1\ngpgkey=https://packages.microsoft.com/keys/microsoft.asc" > /etc/yum.repos.d/vscode.repo'
  3. Update Package Cache and Install
    • For Fedora 22 and later: 
      sudo dnf check-update
sudo dnf install code
    • For older versions: 
      sudo yum check-update
sudo yum install code
    • Wait for installation to complete
  4. Launch VS Code
    • Type code in Terminal and press Enter, or
    • Find "Visual Studio Code" in your application menu
    • Click to open VS Code
Alternative: Download .deb or .rpm Package

You can also download VS Code directly from the website:

  1. Download from Official Website
    • Open your web browser
    • Go to https://code.visualstudio.com
    • Click "Download" and select:
      • .deb for Ubuntu/Debian
      • .rpm for Fedora/Red Hat
  2. Install the Package
    • Navigate to your Downloads folder
    • For Ubuntu/Debian: Double-click the .deb file and click "Install", or run: 
      sudo dpkg -i code_*.deb
sudo apt-get install -f
    • For Fedora: Double-click the .rpm file, or run: 
      sudo dnf install ./code-*.rpm

  1. Install SQLite
    • Open Terminal
    • Run this command: 
      sudo pacman -S sqlite
    • Type Y when asked to confirm
    • Wait for installation to complete

Verify Installation (All Distributions):

  1. Check Version
    • In Terminal, type: 
      sqlite3 --version
    • You should see the SQLite version (typically 3.x.x)
  2. Test SQLite
    • Type: 
      sqlite3
    • You should see the SQLite prompt: sqlite>
    • Try a simple command: 
      SELECT 'SQLite is working!';
    • You should see the message printed
    • Type .quit and press Enter to exit SQLite

Create a Test Database (Optional):

  1. Create a Test Directory
    • In Terminal, run: 
      mkdir ~/test-sqlite && cd ~/test-sqlite
  2. Create a Database
    • Run: 
      sqlite3 test.db
    • Create a simple table: 
      CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT);
INSERT INTO users (name) VALUES ('Alice');
SELECT * FROM users;
    • You should see the data you inserted
    • Type .quit to exit
  3. View the Database File
    • List files in the directory: 
      ls -lh
    • You should see test.db
Useful SQLite Commands

Once you're in the SQLite prompt, here are some helpful commands:

  • .help - Show help information
  • .tables - List all tables in the database
  • .schema - Show the database schema
  • .quit or .exit - Exit SQLite
  • .mode column - Display results in column format
  • .headers on - Show column headers
Visual Studio Code Extensions

VS Code extensions add extra features and functionality to your editor. Here are the recommended extensions for web development:

How to Install Extensions
  1. Open Extensions Panel
    • Open VS Code
    • Press Ctrl + Shift + X to open the Extensions panel, or
    • Click the Extensions icon in the left sidebar (looks like four squares)
  2. Search and Install
    • Type the extension name in the search box
    • Click the "Install" button for the extension you want
    • Wait for installation to complete
    • Some extensions may require VS Code to reload
Essential Extensions (Install Now)
  1. Live Server (ritwickdey.LiveServer)
    • Launches a local development server with live reload
    • Your browser automatically refreshes when you save changes
    • Essential for testing HTML/CSS/JavaScript
    • To use: Right-click an HTML file and select "Open with Live Server"
  2. Prettier - Code formatter (esbenp.prettier-vscode)
    • Automatically formats your code consistently
    • Makes your code easier to read
    • After installing, you can format by pressing Ctrl + Shift + I
    • Enable "Format On Save" in settings for automatic formatting
  3. Auto Rename Tag (formulahendry.auto-rename-tag)
    • Automatically renames paired HTML/XML tags
    • When you rename an opening tag, the closing tag updates too
    • Saves time and prevents errors
  4. HTML CSS Support (ecmel.vscode-html-css)
    • Provides better autocomplete for HTML class and ID attributes
    • Suggests CSS classes from your stylesheets
    • Makes writing HTML faster and more accurate
Helpful Extensions
  1. Path Intellisense (christian-kohler.path-intellisense)
    • Autocompletes file paths in your code
    • Helpful when linking CSS files, images, or other resources
  2. Indent-Rainbow (oderwat.indent-rainbow)
    • Colors indentation levels with subtle colors
    • Makes code structure more visible
    • Helps you spot indentation errors quickly
  3. Error Lens (usernamehw.errorlens)
    • Shows errors and warnings inline in your code
    • Makes debugging easier by highlighting problems immediately

Chromebooks are inexpensive laptops that run Chrome OS, but they are not great for web development. It is possible to install Linux on some Chromebooks, but Google makes it difficult to do this without opening the computer up and making changes to the hardware. This will, at the very least, void your warranty and I honestly do not recommend doing this. I destroyed one Chromebook when I tried. Android phones and tablets are not ideal for web development. While there are some code editors available for Android, the experience is limited compared to using a laptop or desktop computer. If possible, use a Windows, or Linux machine for the best experience.

Apple iPhones and iPads are not ideal for web development. While there are some code editors available for iOS, the experience is limited compared to using a laptop or desktop computer. If possible, use a macOS machine such as an iMac or MacBook for the best experience.
